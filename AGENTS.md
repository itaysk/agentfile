This project is a POC, so you can make breaking changes or suggest big changes if they make sense for the task. If you do, make sure to ask or notify the user.

This project is built using a "docs-driven" approach. The docs (and the UX) are what matter most. If the docs describe a behavior, it should be implemented in code. If a behavior is implemented in code (or observed), it should be documented.

Capitalization rules: use `agentfile` (lowecase) for the YAML file and literal filenames such as `agentfile.yaml`; use `Agentfile` (capitalized) only for the project or product name.
